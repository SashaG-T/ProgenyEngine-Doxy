Detailed Description

template<typename T>
class _FreezableController< T, true >

Template definition for when T is a Controller derived type.

This is a wrapper for any Controller class used to "Freeze" and "Thaw" a Controller. See FreezableController for usage.

Warning
This template should not be used directly. Instead use FreezableController<Controller>.

◆ onFreeze()

template<typename T >
virtual void _FreezableController< T, true >::onFreeze ( )
inlinevirtual

Called when _FreezableController switches to it's frozen state.

Events pushed by GraphicsInterface::pushEvent() will be recieved by other EventListeners before the end of the cycle. The Controller wrapepd by _FreezableController will not receive anymore events until it is thawed.

Note
Controller::getGameObject().getParent() will return Controller::getGameObject() within the scope of this function.

◆ onThaw()

template<typename T >
virtual void _FreezableController< T, true >::onThaw ( )
inlinevirtual

Called when _FreezableController switches to it's thawed state.

Events pushed by GraphicsInterface::pushEvents() will not be recieved by the Controller wrapped by _FreezableController. The Controller wrapped by _FreezableController will only recieve events starting the next cycle. If you want to have something performed on the exact cycle that _FreezableController thaws then look into creating a GameObject that listens for the THAW events.
